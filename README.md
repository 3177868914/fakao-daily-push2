# 📝 法考每日一题自动推送 v2（零Cookie版）

自动抓取微博法考名师的「每日一题」，推送到微信，答案晚间再推送。

## ✨ 特点

- 🚫 **不需要微博Cookie！** 使用公开API，永不过期，零维护
- ⏰ 每天早上 8:00 推送题目，晚上 20:00 推送答案
- 🔄 自动去重，不会重复推送
- 📱 通过 Server酱 推送到微信
- 🎯 支持6位法考名师：柏浪涛、孟献贵、戴鹏、左宁、李佳、杨帆
- 🛡️ 多重降级机制，请求失败自动重试

## 🚀 快速部署（3步搞定）

### 第1步：获取 Server酱 SendKey

1. 打开 https://sct.ftqq.com ，用 GitHub 账号登录
2. 绑定微信（扫码即可）
3. 复制你的 **SendKey**（格式：`SCT305555TC2f49...`）

### 第2步：Fork 本仓库 & 配置 Secret

1. 点击右上角 **Fork** 按钮
2. 进入你 Fork 的仓库 → **Settings** → **Secrets and variables** → **Actions**
3. 添加一个 Secret：

| Name | Value |
|------|-------|
| `SERVERCHAN_SENDKEY` | 你的 SendKey |

**注意：不需要配置 WEIBO_COOKIE！** 🎉

### 第3步：手动测试

1. 进入 **Actions** 标签页
2. 选择 "法考每日一题 - 题目推送"
3. 点击 **Run workflow** → 选择 question → 运行
4. 检查微信是否收到推送 ✅

## 📅 自动运行

| 时间 | 内容 |
|------|------|
| 🌅 早上 8:00 | 推送题目 |
| 🌙 晚上 20:00 | 推送答案 |

## ⚠️ 注意事项

- **Server酱免费版** 每天限5条消息，6位老师可能不够。建议在 `config/teachers.json` 中只保留你关注的2-3位老师
- **微博API限制**：公开接口可获取约90%以上的内容，极少数微博可能遗漏（对每日一题影响极小）
- **GitHub Actions 延迟**：定时任务可能延迟几分钟，属正常现象

## 🔧 自定义

编辑 `config/teachers.json` 可：
- 添加/删除老师
- 修改关键词过滤规则
- 调整时间窗口
